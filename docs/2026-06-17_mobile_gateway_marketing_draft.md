# 离开实验室，算法还在跑

## 主打卖点

AutoBCI 把“离开实验室后仍能监督研究进展”做成一个明确能力：本地 AI 在实验室电脑上持续跑研究闭环，读 Program、排研究方向、跑受限实验、用固定评估器打分、写 ledger；人可以在手机上收到关键状态变化通知，必要时发回少量白名单命令，让电脑暂停、恢复或停止当前研究循环。

这不是普通提醒机器人。普通提醒只告诉你“命令结束了”；AutoBCI 告诉你：

- 当前研究问题是什么
- 正在跑哪条方向
- 为什么选这条方向
- 分数是不是正式结果、候选结果，还是只是一条 smoke
- 产物在哪里
- 哪些动作触发了 Human Gate
- 哪些结论不能被包装成算法提升

一句话版本：

> 你不用守在实验室电脑前。AutoBCI 会让算法继续跑，让证据继续写入 ledger，并把可复盘的研究状态推到你的手机。

## 90 秒 Demo 话术

我现在离开实验室电脑前台，假设人已经不再守着终端或看板。

电脑上 AutoBCI 继续跑一个研究循环任务。它不会偷偷改评估标准，也不会碰原始数据；每一步都写入本地 ledger。

手机收到一条关键状态变化通知：

```text
AutoBCI 研究状态
- 当前 Program：本地 BCI 研究任务
- 当前阶段：research-loop status
- 当前方向：候选 track / 最近执行方向
- 最近结果：smoke / candidate / fixed-evaluator result
- Dashboard：本机只读看板地址
- Report / Ledger：本机可复盘路径

边界：Program / ledger / events / artifacts 才是审计真源。
```

我在手机里问：“现在进展如何？”它不会自由发挥，而是读取真实的：

```bash
autobci status --json
autobci ask "现在进展如何？" --json
```

如果我要控制，也只能发白名单意图，例如 pause、resume 或 stop；控制动作必须显式授权，不能把手机消息当成任意 shell。

这就是 AutoBCI 和普通 coding agent 的区别：coding agent 能改代码，AutoBCI 管住研究闭环。手机入口不是新的主脑，只是把本地研究飞行记录仪延伸到你手上。

## 为什么这是卖点

实验型算法的真实瓶颈不是“多跑一个模型”，而是长任务不可监督、结果不可复盘、边界容易漂移。

AutoBCI 的手机研究链路把这个痛点变成可见能力：

- 研究不停：长实验继续在本地机器或实验室服务器上跑。
- 人不必守着：手机收到关键状态变化通知和报告摘要。
- 控制不过界：手机只能触发白名单动作，不能任意执行 shell。
- 证据不断链：所有入站命令、出站通知、执行结果都继续写审计日志。
- 可迁移：AutoBCI 是第一个垂直样板，后续可以适配药物筛选、材料实验、交易研究、仿真优化、自动化数据清洗等长循环领域。

## 最强反对意见

反对意见：这不就是把日志发到微信吗？

回答：如果只是日志转发，那确实不值钱。关键差异在三层：

1. 发送的不是裸日志，而是从 Program、Research State、固定评估器和 ledger 里抽出的研究状态。
2. 手机端不是任意遥控，而是 token 授权、命令白名单、默认只观察、控制显式授权。
3. 外部微信 / Hermes / OpenClaw / webhook 都只是通道，科研真源仍在 AutoBCI 的本地 artifacts。

## 对外边界

不要承诺“手机完全控制电脑”。这句话危险，而且容易让人误以为是远控软件。

更准确的说法：

> 手机是研究闭环的观察和授权入口，不是无限权限的远程桌面。AutoBCI 只开放状态查询、报告查看和少量白名单控制动作；真正的研究状态仍由本地 Program、ledger、events 和 artifacts 掌握。
